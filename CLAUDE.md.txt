## PROJECT OVERVIEW
Playwright automation for an Oman motor insurance broker.
Reads a customer's renewal data from the broker portal (Tameen) and fills the
matching new-policy form on each insurer's website. Nothing is ever submitted
automatically — every flow STOPS at the review/summary/draft stage so a human
checks it first.

Three insurers are supported:
  • MIC  — Muscat Insurance Company
  • NI   — The New India Assurance Company
  • IRAN — Iran Insurance Company
(A fourth insurer's rows are simply skipped.)

## THE FILES
- common.py    — shared helpers: Tameen reading, MIC + New India + IRAN
                 form-fill, date/plate/body-type utilities. All iran_* helpers
                 and IRAN_* config now live here (moved out of test.py) so the
                 UI / production can import them exactly like MIC/NI.
- production.py — the real daily journey. Runs MIC + New India from the Tameen
                  PAYMENTS page. IRAN is NOT in production yet.
- test.py      — interactive tester for MIC + New India + IRAN off the Tameen
                 PAYMENTS page (you pick channel + row by number each time).
- test_ni.py   — unattended tester for MIC + New India off the Tameen
                 APPLICATIONS page (walks every matching row by itself).
- test_iran.py — unattended tester for IRAN ONLY off the Tameen APPLICATIONS
                 page. Same style as test_ni.py. (IRAN's manual CAPTCHA means
                 the first IRAN record still pauses once for login.)
- app.py       — control-panel UI.
- ni_*.py      — New India body-type / car scraping helpers.

Only ONE of these can run at a time — they share the Chromium profile
(automation_profile), and Chromium allows one window per profile.

## KEY BEHAVIOURS TO KNOW
- Tameen fields are read via the clipboard, so the Tameen tab must be in front
  when reading (each loop calls bring_to_front()).
- IRAN documents (Civil ID + License, front/back) are downloaded from Tameen's
  "Document Details" section and re-uploaded to IRAN. The download uses a
  route-intercept because Tameen's "View" links mint one-time e-locker URLs.
- On IRAN's Plan Details tab, "PAB Cover for Driver Only" and "Pab Cover To
  Passenger" are ALWAYS ticked (mandatory), regardless of Tameen add-ons.
