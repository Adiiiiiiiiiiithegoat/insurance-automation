## 2. THE COMPANY & THE GOAL

Online motor-insurance **broker** based in Oman. Works with **6 insurer companies**.
- **3 insurers** already have **APIs** — fully automated, not my concern.
- **3 insurers** require **manual processing** — this is what I am automating.

**The broker portal is called Tameen** (`mis.tameen.om`). This is where customer
policy data and documents live.

### What staff currently do manually (10–15 min per policy):
1. Read Arabic documents (Mulkiya + License) the customer uploaded
2. Cross-check document data against customer-submitted data
3. Log into the insurer's website
4. Fill in a form field by field
5. Handle page reloads during the form
6. Approve the policy
7. Download the resulting PDF
8. Upload the PDF back to the broker portal
9. Send the PDF to the customer on WhatsApp

**Volume:** 30–70 policies/day across the 3 manual insurers.
**Goal:** Reduce staff work to **only flagged exceptions** (where document data
does not match submitted data). Everything else runs automatically.

## 3. CURRENT STATE — WHAT ALREADY WORKS

The OrangeHRM demo placeholder has been **completely replaced** with the **real
MIC (Muscat Insurance Company) insurer website**. The script is now end-to-end
on real systems for Muscat Insurance. Here is exactly what it does today:

### Tameen (broker portal) side — REAL site, working:
1. Opens `https://mis.tameen.om/dashboard/login` and **waits for manual login**
   (Tameen requires OTP — this is the only manual pause, then ENTER resumes automation).
2. Clicks the **PAYMENTS tile** on the dashboard.
3. Clicks the **"Payments by Channel"** red button (top-right of the Payments page).
4. Reads all channel tiles from **both sections** ("Payment Done Cases" and
   "Payment Done Document Pending Cases"), shows them numbered with their record
   counts, and **asks the staff member to pick a channel**.
5. Reads the records table, **filters to "Muscat Insurance" rows only**, lists
   them numbered, and **asks which row to open**. Typing `0` here goes back to
   the channel select. Typing `q` at any continue prompt quits.
6. Clicks the **eye icon** on the chosen row.
7. **Reads these fields** from the opened record:
   First Name, Last Name, License ID, Product Name, Previous Expiry,
   Vehicle Number (e.g. "B S-4788"), Sum Insured, Total Premium, Seats.

### MIC (Muscat Insurance Company) insurer — REAL site, working:
8. **Logs in automatically** using credentials from a `.env` file (no OTP on MIC).
   If the session from a previous run is still alive (saved profile), skips login.
9. Home → **Policy tile** → **Create** button.
10. Reads "Product Name" from Tameen and selects **Comprehensive or Third Party**
    in the Policy Type dropdown, then clicks **Create** inside the popup, then
    clicks **OK** on the confirmation dialog.
11. Types the **License No** → clicks **"Get Licence Information" → "Get Record"**
    (MIC auto-fills: Insured Name, DOB, Address, Lic Issue Date, Lic Expiry Date).
12. Fills **Policy Information**: Cust Code (21252), Insured Name, Period From
    (calculated from the previous expiry), Address (Muscat), Mobile (99435202),
    Mulkiya Type (Renewal).
13. Fills **plate number**, selects **plate code** via popup-LOV (handles both
    single-letter 'A' and double-letter 'B S' plates), clicks
    **"Get Vehicle Information" → "Get Record"** (waits ≥8 s for vehicle data).
14. In Vehicle Information: sets **Geo Area = UAE**, switches
    **"Road Side Assistance - Silver"** from No to **Yes** (via real double-click
    on the editable grid), fills **Seats** (read live from Tameen), and if
    Comprehensive fills **Sum Insured**.
15. Clicks **Calculate**, clicks **OK** on the "unsaved changes" APEX dialog,
    then reads **"Net Prem Incl. VAT"** from the Premium Details panel and
    **compares it to Tameen's "Total Premium"**. Tolerance: **1 OMR**.
    - Match → prints "PREMIUMS MATCH".
    - Mismatch → prints a large warning banner with the difference; policy is left
      as Draft and staff must investigate before doing anything else.
16. The script **stops at Draft** — no auto-approve (deliberate).

### Per-record loop:
After each record (success or flagged error) the script prompts ENTER to
**reset both tabs** (MIC back to home, Tameen back to the Payments page) and
continue to the next policy. Type `q` to quit and close the browser.

### Config values (top of main.py):
| Name | Value | Notes |
|---|---|---|
| `CUSTOMER_CODE` | `21252` | Fixed for all policies |
| `FIXED_ADDRESS` | `Muscat` | Fixed |
| `FIXED_MOBILE` | `99435202` | Fixed |
| `MULKIYA_TYPE` | `Renewal` | Fixed |
| `FIXED_GEO_AREA` | `UAE` | Despite the name, the form shows "UAE" not "Oman" |
| `PREMIUM_TOLERANCE` | `1.0` OMR | Treat premiums within 1 OMR as a match |
| `ADDL_BENEFIT_DESC` | `ROAD SIDE ASSISTANCE - SILVER` | The exact row text in the grid |
| `MIC_HOME_URL` | `https://portal.muscatinsurance.com:444/ords/r/mic/mic/home` | |

### Notable techniques already in the code (keep these — they work):
- `launch_persistent_context` with a saved `automation_profile` folder so login
  sessions/cookies persist between runs.
- `slow_mo=450` to make actions visible and stable.
- `ignore_https_errors=True` because MIC uses a custom cert on port `:444`.
- Clipboard read permissions enabled for Tameen's copy-icon strategy.
- `mic_fill_by_label()` types character-by-character (`delay=20`) so APEX `onChange` fires.
- Every interaction has **multiple fallback strategies** wrapped in try/except.
- `wait_for_apex()` waits for APEX's AJAX overlay to disappear after every APEX action.
- `mic_handle_popup_lov()` drives the APEX popup-LOV modals (Plate Code field):
  finds the matching row with JS, then does a REAL Playwright click (APEX ignores
  synthetic JS clicks).
- `mic_accept_confirm_dialog()` handles the in-page APEX HTML dialog that appears
  after clicking Calculate. This is NOT a native browser dialog; it is a DOM
  element with a green OK button. See `errorlog.md.txt` for the full lesson.
- `mic_click_button()` only clicks VISIBLE buttons and skips hidden/covered ones.
- Native browser dialogs (beforeunload "Leave site?") are auto-accepted via
  `page.on("dialog", lambda d: d.accept())` registered on BOTH pages at startup.
- `.env` file for MIC credentials (`MIC_USERNAME`, `MIC_PASSWORD`).

### Known rough edges:
- `read_field()` relies partly on the clipboard, which can be flaky.
- The Tameen field labels for Vehicle Number, Sum Insured, Total Premium, and Seats
  are **guessed** — they need confirming against the real record page.
- `FILTER_TO_DATE` and date-range filtering are no longer used (replaced by the
  "Payments by Channel" flow).

## 4. THE FULL TARGET WORKFLOW (where this is going)

```
1.  Staff identifies a policy to process on Tameen (the ONLY manual trigger)
2.  Script navigates Tameen → Payments → Payments by Channel →
    choose channel → choose row → open record
3.  Script scrapes all relevant text fields from the record
4.  Script opens the Mulkiya (image/PDF) → AI OCR → extract Arabic fields
    (plate, chassis, civil ID, owner name, vehicle details, etc.)
5.  Script opens the License → AI OCR → extract driver/license fields
6.  CROSS-VERIFY document values vs. Tameen submitted values
        match     → continue automatically
        mismatch  → STOP, log the exact mismatch, alert staff, do nothing else
7.  Fill the MIC insurer form (22-step flow — already working for Muscat Insurance)
8.  Approve the policy, download the PDF (save named by policy number)
9.  Return to Tameen, upload the PDF + any extra details to confirm the policy
10. Send the PDF to the customer's WhatsApp number
```

## 9. CURRENT CODE — STRUCTURE SUMMARY

Single file: `main.py` (Python, `playwright.sync_api`), organised as:

- **CONFIG block:** MIC URL, fixed field values, policy type strings, premium
  tolerance, step pause duration. Credentials loaded from `.env` via `python-dotenv`.
- **Shared helpers:** `read_field()` (clipboard + XPath fallbacks),
  `keystroke_fill()`, date parsing (`parse_tameen_date`, `compute_period_from`),
  `split_plate()` (splits "B S-4788" into code "B S" and number "4788").
- **APEX helpers:** `wait_for_apex()`, `mic_fill_by_label()`, `mic_select_by_label()`,
  `mic_click_button()` (visible-only), `mic_accept_confirm_dialog()` (in-page APEX
  HTML dialog), `mic_handle_popup_lov()` (real-click popup-LOV for Plate Code),
  `mic_set_cust_code()` (type + Enter, handles any popup, verifies).
- **MIC flow helpers:** `mic_login_if_needed()`, `mic_click_tile()`,
  `mic_open_policy_create()`, `mic_choose_policy_type_and_create()`,
  `mic_get_licence()`, `mic_fill_policy_info()`, `mic_get_vehicle()`,
  `mic_enable_addl_benefit()`, `mic_fill_vehicle_info()`, `mic_calculate_and_check()`,
  `read_premium()`.
- **Tameen helpers:** `tameen_go_to_payments()`, `tameen_click_payments_by_channel()`,
  `tameen_select_channel()` (reads ALL tiles from BOTH sections with JS, numbered menu),
  `tameen_select_and_click_eye()` (reads rows, filters Muscat Insurance, numbered menu,
  supports going back to channel select by typing 0).
- **Reset helpers:** `tameen_reset_to_payments()` (in-app back-navigation),
  `mic_reset_to_home()` (goto MIC home URL).
- **Main flow:** opens 2 tabs (Tameen + MIC), registers native-dialog auto-accept on
  both, pauses for manual Tameen login/OTP (ONCE), then enters the per-record loop:
  Tameen navigate → read fields → MIC fill → Calculate → premium check → staff reviews
  → ENTER to reset + continue, or 'q' to quit.
